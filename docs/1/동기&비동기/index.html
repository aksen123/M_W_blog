동기,비동기 개념과 차이

동기(synchronous) 방식​

요청과 결과가 동시에 일어나는 방식으로 요청을 보낸 후 응답을 받아야 다음 동작이 진행된다 (작성된 순서대로 처리하는 방식)

  • 서버로 요청을 보내면 요청에 대한 응답을 받기 전까지 다음 동작들은 진행 불가 ( 커피머신 이라고 생각하면 됨 A작업이 끝날때 까지 다른 B작업 시작 X )

  • 장점 : 설계가 간단하며 직관적

  • 단점 : 요청 결과가 반환되기 전까지 대기 해야함

비동기(Asynchronous) 방식​

요청과 결과가 동시에 일어나지 않는 방식 (작성된 순서와 상관없이 처리)

  • 서버에 보낸 요청에 대한 응답을 기다리지 않고 다른 요청을 서버에 보낼수 있다 ( A작업을 하면서 B작업 시작 O)

  • 장점 : 요청에 대한 결과가 반환되기 전에 다른 작업을 수행 할 수 있어 자원을 효율적으로 사용할 수 있다.

  • 단점 : 동기 방식보다 설계가 복잡하다.
